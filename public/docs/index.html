    $LOOM-colors-map-default: (
  'black': #000,
  'white': #fff
) !default;

    Description

    The central, default, global color list.

    Retrieve theme colours defined here with get-color(name).

    Type

    Map

    Example

    Using a colour map

    $LOOM-colors-map-default: (
  'brand-primary': #ed3d68,
  'brand-secondary': #30bbba,
  'white': #fff,
  'black': #000,
  'meta': #e4e4e4
)

    Using a colour map with predfined variables

    $color-pink: #ed3d68;
$color-green: #30bbba;
$color-light-gray: #e4e4e4

$LOOM-colors-map-default: (
  'brand-primary': $color-pink,
  'brand-secondary': $color-green,
  'white': #fff,
  'black': #000,
  'meta': $color-light-gray
)

    mq

    @mixin mq($from, $until, $and, $media-type) { ... }

    Description

    Media Query mixin

    Lovingly repurposed from https://github.com/sass-mq/sass-mq

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $from

    (false) - One of $LOOM-mq-breakpoints

    		String or Boolean—none
    $until

    (false) - One of $LOOM-mq-breakpoints

    		String or Boolean—none
    $and

    (false) - Additional media query parameters

    		String or Boolean—none
    $media-type

    ($LOOM-mq-media-type) - Media type: screen, print…

    		String—none

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    .element {
  @include mq($from: mobile) {
    color: red;
  }
  @include mq($until: tablet) {
    color: blue;
  }
  @include mq(mobile, tablet) {
    color: green;
  }
  @include mq($from: tablet, $and: '(orientation: landscape)') {

    mixins

    button

    @mixin button() { ... }

    Description

    This mixin creates a button, along with some basic styles.

    It has several arguments, each with basic defaults, that can be used to customise the button to your needs.

    By default the hover state of a button is a slight darkening of the background-color, color and border-color. If this is not appropiate then all the hover colours can be overridden.

    The defaults for the arguments (the default defaults) can be overridden on a project basis, so that new buttons can be generated without supplying project-specific arguments each time.

    Parameters

    None.

    Requires

    • [function] get-transition
    • [variable] LOOM-button-background-color
